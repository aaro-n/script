RSSHub 生成器 - 参数说明文档

理解每个参数的用途，释放 RSSHub 的全部潜力。

核心概念： 大部分高级参数的值都是 CSS 选择器。它是一种用来在 HTML 文档中定位元素的强大语法。如果你不熟悉，强烈建议先阅读 MDN CSS 选择器教程。

基本配置

host

你的 RSSHub 服务部署地址。可以是官方地址，也可以是你自己搭建的服务的地址。

示例：https://rsshub.app

url

你希望为其制作 RSS Feed 的目标网页的完整 URL。

示例：https://sspai.com/matrix

key

如果你在自建的 RSSHub 服务中配置了访问密钥 (Access Key) 以防止滥用，请在此处填写。这是一个可选参数。

示例：my_secret_key_123

limit

限制最终生成的 RSS Feed 中包含的条目数量。值为正整数，可选参数。

示例：15 (表示最多获取 15 篇文章)

mode=fulltext

启用全文抓取模式。勾选后，RSSHub 会尝试访问每个条目的链接，并抓取其页面的正文内容填充到 RSS 的描述中。

  • 这会显著增加服务器的请求次数和处理时间。
  • 当目标网站有反爬虫措施时，可能会失败。

高级路由参数 (routeParams)

title

定义整个 RSS Feed 的标题。如果不填，RSSHub 会默认使用目标网页的 <title> 标签内容。

示例 (CSS 选择器):h1.page-title

item

【最重要参数】用于定位文章列表的范围。你需要提供一个 CSS 选择器，它能够选中 **包含每一篇文章所有信息（标题、链接、摘要等）的外层容器**。

示例 (CSS 选择器):article.post-item

itemTitle, itemLink, itemDesc, itemPubDate, itemContent

这些参数用来从每个 item 容器内部提取具体信息。它们的选择器是 **相对于 item 元素** 的。

  • itemTitle: 文章标题。
  • itemLink: 文章链接，通常指向一个 <a> 标签。
  • itemDesc: 文章摘要或描述。
  • itemPubDate: 文章发布日期。RSSHub 会智能解析多种日期格式。
  • itemContent: 直接从列表页提取文章内容，作为 fulltext 的替代方案。
示例 (CSS 选择器):
  • itemTitle: h2.entry-title a
  • itemLink: .post-wrapper > a
  • itemDesc: p.summary
  • itemPubDate: span.date

属性提取参数 (...Attr)

当需要的信息不在元素的文本中，而是在其 **属性**（如 src, href, data-*, datetime）里时，就需要使用这些参数。

itemLinkAttr

指定从 `itemLink` 选中的元素上提取哪个属性作为链接。默认为 href。

场景：链接在自定义属性 data-link 中。
示例：data-link

itemDescAttr

指定从 `itemDesc` 选中的元素上提取哪个属性作为描述。

场景：想用文章封面图作为描述。itemDesc 设为 img.cover。
示例：src (提取图片的源地址)

itemPubDateAttr

指定从 `itemPubDate` 选中的元素上提取哪个属性作为日期。

场景：日期在 <time> 标签的 datetime 属性中。
示例：datetime

其他参数

encoding

当目标网站是老旧网站，使用非 UTF-8 编码（如 GBK, GB2312）导致抓取内容乱码时，可在此手动指定正确的编码。

示例：gbk 或 gb2312
